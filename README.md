## Задание 2: Имена

**Формулировка:**
Создать сущность "Имя" с параметрами: Фамилия, Личное имя, Отчество. Имя должно приводиться к строковому виду в традиционном формате. Если какой-либо параметр не задан, он не должен учитываться при выводе.

Создать имена:
- Клеопатра (только имя)
- Пушкин Александр Сергеевич (полное имя)
- Маяковский Владимир (фамилия и имя)

**Решение:**
Класс com.example.Name обрабатывает различные комбинации параметров через перегруженные конструкторы и формирует строковое представление.

**Изменения:**
1. Добавлена проверка на null во всех конструкторах  
Проблема: при передаче null в конструктор возникало NullPointerException  
Исправление: добавлены операторы ```(param != null) ? param : ""```  
Было:  
```this.lastName = lastName;```  
Стало:  
```this.lastName = (lastName != null) ? lastName : "";```  
2. Исправлено создание имени "Клеопатра" в main.java  
Проблема: использовался неправильный конструктор  
Исправление: правильный конструктор с одним параметром  
Было:  
```Name name1 = new Name("", "Клеопатра", "");```  
Стало:  
```Name name1 = new Name("Клеопатра");```

## Задание 4: Города с путями

**Формулировка:**
Создать сущность "Город" с названием и набором путей к другим городам. Каждый путь представляет собой пару "город-стоимость". Город должен возвращать текстовое представление с названием и списком связанных городов в формате "название(стоимость)".

Реализовать схему городов:
- A → B(5), F(1), D(6)
- B → A(5), C(3)
- F → B(1), E(2)
- C → B(3), D(4)
- D → C(4), E(2), A(6)
- E → F(2), D(2)

**Решение:**
Класс com.example.City использует параллельные списки для хранения связей между городами и соответствующих стоимостей.

**Изменения:**
1. Добавлена проверка на null для названия города  
Проблема: передача null приводила к ошибкам при выводе  
Исправление: пустое название заменяется на пустую строку  
Было:  
```this.name = name;```  
Стало:  
```this.name = (name != null) ? name : "";```  
2. Добавлена проверка на отрицательную стоимость пути  
Проблема: можно было добавить путь с отрицательной стоимостью  
Исправление: добавлено условие ```cost >= 0```  
Было:  
```if (city != null && !connectedCities.contains(city))```  
Стало:  
```if (city != null && !connectedCities.contains(city) && cost >= 0)```

Задачи:
1. Создать несколько дробей
2. Показать примеры всех операций

Улучшен метод toString()

```return numerator + "/" + denominator;```
